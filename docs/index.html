iReview
Install Features GitHub Releases

Review your AI's diffs in a browser.

Point iReview at any local git repository, pick any combination of recent commits + uncommitted work, review the diff like a GitHub PR, and copy a structured Markdown summary back to your coding agent.

Install View on GitHub

Install

Homebrew RECOMMENDED

macOS & Linux — skips Gatekeeper friction.

brew tap suhothayan/tap
brew install ireview
ireview

Binary

Download from Releases, then run. 

./ireview /path/to/repo

From source

Vite + Express, hot reload.

git clone https://github.com/suhothayan/iReview
cd iReview && npm install
IREVIEW_REPO=/path npm run dev

Features

Pick any selection

Recent commits + staged + unstaged in one picker — combined into a single diff. Pick A and C, B comes along if it sits between.

GitHub-style diff viewer

Unified diff with hunk headers, file tree sidebar, sticky per-file headers, comment-count badges, light + dark themes.

Click to comment

Click a line to leave a typed comment (must-fix / suggestion / note). Shift-click another line on the same side to extend a multi-line range.

Mark files reviewed

Per-file checkboxes plus tri-state cascading folder ticks — mark a whole directory reviewed in one click.

Two view modes

Single-file with Prev/Next, or scroll-all GitHub-style with sidebar that follows scroll position.

Copy review

Numbered, structured Markdown lands on your clipboard with one click. Paste into your AI agent's chat to apply fixes.

Output format

The exported review is structured Markdown optimised for pasting into AI agent chats — numbered for easy reference (you can ask the agent to "address comment #2"). 

I reviewed your code and have the following comments. Please address them.

Comment types: MUST FIX (must be addressed), SUGGESTION (improvements), NOTE (observations)

1. [MUST FIX] - `src/db.ts:42-50` - this transaction can deadlock under contention
2. [SUGGESTION] - `src/auth.ts:12` - add a more specific error message
3. [SUGGESTION] - `src/auth.ts:50` - prefer Result<> over throwing here
   the calling layer would rather branch on a typed result than wrap a try/catch
4. [NOTE] - `src/auth.ts` - consider extracting this into a dedicated module
